# Contributing to VibeParts

Welcome! We're excited that you're interested in contributing to VibeParts! Below are some guidelines to get you started.

## Code of Conduct
We expect all contributors to adhere to the following code of conduct:
- Be respectful and kind to others.
- Be open to feedback and new ideas.
- No harassment, discrimination, or any form of abusive behavior.

## Development Setup
To get started with developing for VibeParts, follow these steps:
1. **Clone the repository**: 
   ```sh
   git clone https://github.com/toan1908/VibeParts.git
   cd VibeParts
   ```
2. **Install dependencies**: 
   ```sh
   npm install
   ```
3. **Run the project**: 
   ```sh
   npm start
   ```

## Pull Request Process
1. **Fork the repository** to your account.
2. **Create a new branch** for your feature or bug fix:
   ```sh
   git checkout -b my-feature
   ```
3. **Make your changes** and commit them with clear messages:
   ```sh
   git commit -m "Add a new feature"
   ```
4. **Push to your fork**:
   ```sh
   git push origin my-feature
   ```
5. **Open a pull request** from your feature branch to the `main` branch on the original repo.  
   Make sure to include a description of the changes and why they are necessary.

Thank you for contributing to VibeParts! Happy coding!
